# Ralph Progress Log — Konstruq V3
Started: 2026-02-28

## Codebase Patterns
- Database: Neon Postgres via drizzle-orm/neon-http. Schema at src/lib/db/schema.ts. Push via `npx drizzle-kit push`.
- Auth: Clerk. All API routes use `const { userId } = await auth()` then `resolveAppUserId(userId)` from src/lib/db/app-user.ts.
- API routes: Next.js 16 App Router. Route handlers export async functions (GET/POST/DELETE/PATCH). Return NextResponse.json().
- UI: shadcn/ui components in src/components/ui/. Dark theme. Amber/cyan accent colors.
- AI: Mistral via @ai-sdk/mistral. `generateText` for dashboard generation, `streamText` for chat.
- Dashboard: AI-generated via POST /api/dashboard/generate. DashboardClient renders KPI cards + DynamicChart.
- Chat: useChat from ai/react wrapped in ChatRuntimeProvider at layout level. Tools in _lib/data-tools.ts.
- Data: uploadedDatasets stores parsed CSV/XLSX rows as JSONB in sheets column.
- Two separate ChartSpec types: chat (src/lib/charting/spec.ts) and dashboard (src/lib/dashboard/types.ts).
- Chat message rendering: src/components/chat/message-renderer.tsx parses ```chart / ```table / ```kpi blocks.
- Chat chart rendering: ChartBlock → ChartFromSpec (Recharts-based).
- Dashboard chart rendering: DashboardClient → DynamicChart.
- Shared dashboard generation utils: src/lib/dashboard/generate.ts (fetchUserDatasets, callDashboardLLM, parseJsonResponse).
- Env vars: .env.local (DATABASE_URL, MISTRAL_API_KEY, CLERK_SECRET_KEY etc.)
- Styling: Tailwind v4 with tw-animate-css. Global styles in globals.css.
- Codex CLI: use `codex exec --model gpt-5.3-codex-xhigh --yolo --output-last-message <file> - < <prompt> > <log> 2>&1`
