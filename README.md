# Konstruq ‚Äî AI-Powered Construction Analytics Platform

Konstruq is an AI-powered data analytics platform for the construction industry. Users upload project data (CSV/XLSX), ask questions in natural language, and get interactive charts, KPIs, and dashboards ‚Äî all generated by an AI agent with deep construction-domain knowledge.

Integrates **Procore** (project management) and **Sage Intacct** (ERP/financial) into a unified analytics workspace.

## Tech Stack

| Layer | Technology | Role |
|-------|-----------|------|
| Framework | Next.js 16 (App Router) | Full-stack: dashboard UI + all API routes in one codebase |
| Language | TypeScript + Zod | End-to-end type safety; Zod validates AI tool call parameters at runtime |
| AI | Mistral Large + Vercel AI SDK | LLM-powered chat agent with streaming responses and multi-step tool calling |
| Database | Neon (PostgreSQL) + Drizzle ORM | Serverless Postgres; datasets stored as multi-sheet JSONB; type-safe queries |
| Auth | Clerk | Authentication, session management, user identity resolution |
| Charts | Recharts | Renders AI-generated chart specs as interactive React components |
| UI | shadcn/ui + Tailwind CSS 4 | Component library (cards, dialogs, sheets, tabs, tables, popovers) |
| Data Parsing | PapaParse + SheetJS | Client-side CSV/XLSX parsing into structured sheet data |

## AI System

### Dual Agent Architecture

Konstruq runs two specialized AI agents, both powered by Mistral Large with streaming (`streamText`) and multi-step tool calling (`maxSteps: 8`):

- **Chat Analytics Agent** (`/api/chat`) ‚Äî Full conversational agent for exploratory analysis, forecasting, and earned value insights. Equipped with 5 data tools (list/schema/query/aggregate/search) that let it autonomously explore uploaded datasets.
- **Dashboard Builder Agent** (`/api/custom-dashboards/[id]/chat`) ‚Äî Focused agent that generates one chart or KPI per response, scoped to a specific project's datasets. Operates inside the custom dashboard sidebar chat.

### AI Tool System

The agents have access to 5 Zod-validated tools with a mandatory workflow (list ‚Üí schema ‚Üí query):

| Tool | Purpose |
|------|---------|
| `listDatasets` | Discover available files, sheets, columns, row counts |
| `getDatasetSchema` | Inspect column types and sample values |
| `queryDatasetRows` | Fetch rows with filtering, pagination, column projection |
| `aggregateColumn` | Compute sum/avg/min/max/count/distinct with pre-filtering |
| `searchDatasets` | Cross-dataset full-text search by value |

### Chart Selection Engine

AI-generated charts pass through a multi-stage normalization pipeline (`normalizeChartSpec`):

1. **Intent detection** ‚Äî Infers question type (trend / comparison / forecast / composition / etc.)
2. **Domain awareness** ‚Äî Applies domain-specific rules (construction, finance, healthcare, retail)
3. **Policy enforcement** ‚Äî 50+ rules (e.g., pie only ‚â§6 categories, scatter requires two numeric axes)
4. **Automatic fallbacks** ‚Äî Pie‚Üíbar, scatter‚Üíbar, large datasets‚Üítop-N + "Other"
5. **Source validation** ‚Äî Verifies chart keys map to cited dataset columns

### Dashboard Generation

Three variant-specific pipelines, each with tailored system prompts:

- **Executive** ‚Äî High-level business KPIs + 4‚Äì8 insight charts
- **Project Controls** ‚Äî Schedule tracking, SPI/SV trends, equipment utilization
- **Financial Controls** ‚Äî Budget vs actual, EVM S-curves, cash flow, cost breakdowns

Includes token budgeting (20K cap), content-addressable server-side caching, and data-version-aware invalidation.

## Database Schema

Managed by Drizzle ORM, pushed to Neon via `npm run db:push`:

- **users** ‚Äî Synced from Clerk (clerkId, email, name)
- **projects** ‚Äî User-created workspace projects
- **dataConnections** ‚Äî Procore/Sage credentials per user
- **syncedProjects** ‚Äî Cached API responses from Procore
- **uploadedDatasets** ‚Äî User CSV/XLSX uploads (multi-sheet JSONB)
- **customDashboards** ‚Äî User-created dashboards per project
- **dashboardWidgets** ‚Äî Chart/KPI widget configs (JSONB) per dashboard
